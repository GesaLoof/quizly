# Quizly

Quizly turns any YouTube video into a 10-question multiple-choice quiz. Paste in a video URL, and the app downloads the audio, transcribes it with Whisper, and uses Google's Gemini to generate quiz questions based on the content.

## Features

- JWT authentication with HttpOnly cookies (access + refresh tokens)
- Token blacklisting and rotation on logout/refresh
- YouTube audio extraction via yt-dlp
- Speech-to-text transcription via OpenAI Whisper
- AI-generated quizzes via the Google Gemini API
- Quizzes are scoped per user — each user has their own quiz history for a video
- Full CRUD on quizzes (list, retrieve, update, delete), restricted to the owning user

## Tech Stack

- Django / Django REST Framework
- djangorestframework-simplejwt
- yt-dlp
- OpenAI Whisper
- Google Gemini API (`google-genai`)

## Requirements

Before installing Python dependencies, make sure the following are available on your system:

- Python 3.13+
- [ffmpeg](https://ffmpeg.org/) (required by Whisper for audio processing)
  ```bash
  brew install ffmpeg        # macOS
  sudo apt install ffmpeg    # Ubuntu/Debian
  ```
- [Node.js](https://nodejs.org/) (required by yt-dlp for YouTube extraction)
  ```bash
  brew install node          # macOS
  sudo apt install nodejs    # Ubuntu/Debian
  ```

## Setup

1. Clone the repository and navigate into the project folder.

2. Create and activate a virtual environment:
   ```bash
   python3.13 -m venv venv
   source venv/bin/activate    # macOS/Linux
   venv\Scripts\activate       # Windows
   ```

3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

4. Create a `.env` file (see .env.example) in the project root with the following variables:
   ```
   SECRET_KEY=your_django_secret_key
   DEBUG=True
   GEMINI_API_KEY=your_gemini_api_key
   ```

   Get a Gemini API key at [aistudio.google.com](https://aistudio.google.com).

5. Run migrations:
   ```bash
   python manage.py makemigrations
   python manage.py migrate
   ```

6. Start the development server:
   ```bash
   python manage.py runserver
   ```

The API will be available at `http://localhost:8000/`.

## API Overview

### Auth

| Method | Endpoint                   | Description                          |
|--------|----------------------------|---------------------------------------|
| POST   | `/api/registration/`       | Create a new user account |
| POST   | `/api/login/`              | Log in with username & password, sets JWT cookies |
| POST   | `/api/token/refresh/`      | Refresh the access token using the refresh token cookie |
| POST   | `/api/logout/`             | Blacklist the refresh token and clear cookies |

### Quizzes

| Method | Endpoint                | Description                                  |
|--------|-------------------------|-----------------------------------------------|
| POST   | `/api/quizzes/`         | Generate a quiz from a YouTube URL            |
| GET    | `/api/quizzes/`         | List all quizzes belonging to the current user |
| GET    | `/api/quizzes/{id}/`    | Retrieve a single quiz (owner only)           |
| PATCH  | `/api/quizzes/{id}/`    | Update a quiz's title/description (owner only) |
| DELETE | `/api/quizzes/{id}/`    | Delete a quiz (owner only)                    |

**Example — generate a quiz:**
```json
POST /api/quizzes/
{
    "url": "https://www.youtube.com/watch?v=example"
}
```

**Example response:**
```json
{
  "id": 1,
  "title": "Quiz Title",
  "description": "Quiz Description",
  "created_at": "2026-06-18T12:34:56.789Z",
  "updated_at": "2026-06-18T12:34:56.789Z",
  "video_url": "https://www.youtube.com/watch?v=example",
  "questions": [
    {
      "id": 1,
      "question_title": "Question 1",
      "question_options": ["Option A", "Option B", "Option C", "Option D"],
      "answer": "Option A"
    }
  ]
}
```

## Notes

- Videos are transcribed once and cached — if the same YouTube URL is submitted again, the existing transcript is reused instead of re-downloading and re-transcribing.
- Each user gets their own `Quiz` record per video, so quiz history is kept separate per user even for the same video.
- The Gemini API has a free tier, but rate limits are modest. For development, keep an eye on usage at [ai.dev/rate-limit](https://ai.dev/rate-limit).
